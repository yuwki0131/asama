# 兵糧・補給

## 基本方針

防衛側と攻撃側で兵糧の保持方法が異なります。

## 防衛側

### 食料保存

防衛側の食料は蔵にのみ保存します。

利用可能兵糧は次の合計です。

```text
本丸から通行可能な経路で接続された未破壊の蔵の食料
```

道路は不要です。敵兵の位置は接続判定に含めません。

### 消費順

本丸から遠い蔵から順に消費します。

### 接続判定

- 約8〜12秒のseed付きランダム周期
- 門の開閉直後には即時再計算しない
- 判定断面の結果を次回まで保持

### 補給荷車

- 出現時点では兵糧に含まれない
- プレイヤーが直接操作
- 蔵へ到達すると荷車が消滅
- 積載食料を蔵へ加算
- 蔵の空き容量が積載量未満なら格納完了できず、荷車はその場に残る
- 破壊時は積載食料全損

### 蔵

- 占領不可
- 敵は破壊のみ可能
- 破壊時、保存食料全損

## 攻撃側

攻撃側に蔵、本陣、補給地点、接続経路はありません。

```text
攻撃側兵糧 = 生存している全補給荷車の残食料合計
```

荷車の位置は問いません。城内に存在しても利用可能です。

- 荷車はプレイヤー操作ユニット
- 破壊時は積載分全損
- 食料ゼロの荷車は消滅可能
- 全荷車合計がゼロになると敗北

## 補給荷車のシナリオ設定

次をシナリオごとに設定可能にします。

- 所属勢力
- 出現地点
- 出現時刻
- 出現条件
- 出現回数
- 積載量
- 護衛編成

ゲーム共通の積載量上限は設けません。

## 市場による食料購入

防衛側:

- 本丸から市場へ接続が必要
- 本丸から接続された蔵に空き容量が必要
- 本丸に近い蔵から順に格納
- 荷車は生成しない

攻撃側にはMVPで市場利用ルールを設けません。攻撃側兵糧の追加はシナリオで補給荷車を出現させて表現します。

## 兵糧消費処理順

1. 接続状態をスナップショット取得
2. 到着済み防衛側荷車を蔵へ格納
3. 経過期間の必要食料を計算
4. 防衛側は遠い蔵から消費
5. 攻撃側は荷車プールから消費
6. 不足した側を敗北判定
