# Release 2.0 スコープ合意書

2026-07-07 合意。1.0は `v1.0` タグ(667cb5a)で凍結。1.0の残バグは後続フェーズまたは本リリースで回収する。

## 2.0のテーマ

ビジュアル大改革。1.0とは決定的に異なるアート表現のゲームにする。4本柱:

1. **高低差表現** — 平城一択から、平山城・山城が成立する基盤へ
2. **ユニットのキャラクターアニメーション** — 歩行・戦闘がフィールド上で動く
3. **自然・植生のアニメーション** — 風に揺れる樹木、水面
4. **既存ビジュアルの全面エンハンス** — カクカクした移動、接合部・配置のいびつさの抜本解消

## 合意済み仕様(Q&A確定分)

| 項目 | 決定 |
|---|---|
| 完成物 | 山城(または平山城)の新シナリオ1本をショーケースとして新規制作。既存3シナリオは新基盤上で動作+ビジュアル刷新(地形はフラット維持) |
| 高低差の効果範囲 | ビジュアル+移動(崖=通行不可、坂のみ通行、登坂コスト)+高所の射程・ダメージボーナス。**射線遮蔽は3.0へ** |
| 高さの粒度 | 離散4段(0〜3)。連続高さは将来課題 |
| 曲輪と石垣 | 曲輪の段差=「石垣張りの地形段差」として高低差システムに統合。自然の崖は岩肌スキン、城郭部は石垣スキン |
| アニメ対象 | 全ユニット種(足軽・弓・鉄砲・騎馬・エンジニア・荷車)。エンジニア正式スプライト化を吸収 |
| アクション | idle / walk / attack 必須、death は簡易(倒れ1〜2フレーム+フェード)。建物に旗のなびき・煙の生活感アニメ |
| 方向数 | 8方向(Blenderカメラ回転で量産) |
| 制作方式 | 統括に一任(腹案: リグ付きローポリ+キーフレーム→スプライトシート焼き出し。質感維持は実レンダー比較で確定) |
| 植生・水面 | 手法は統括に一任(腹案: シェーダーベースの揺れ+水面ノイズスクロール。フレーム方式と実験比較) |
| 戦闘エフェクト | **含める**(矢の軌跡・鉄砲の煙と閃光・打撃ヒット・死亡演出)。2.0の本丸。難航時のみ別途調整 |
| 環境演出 | **2.0では含めない**(雲影・環境色調・時間帯表現は後続バージョンの課題) |
| 1.0残課題 | 吸収する: ローディング表示、シナリオ選択UI、エンジニアスプライト |

## 受け入れ基準(統括の自己判定 → 満たしたら最終確認を依頼)

1. E2E 4層(スモーク/機能/オートプレイ/モンキー)すべて緑
2. 標準ズーム・最大ユニット数で 60fps 維持
3. フォールバックスプライト ゼロ
4. 全シナリオのビジュアル回帰スクリーンショット一式を成果物に添付
5. 山城シナリオがオートプレイで勝利・敗北の両方成立

## 進め方の契約

- **非ブロッキング報告**: マイルストーンごとに画像付き報告を置くが、統括は返答を待たず走り続ける。ユーザーの口出しは任意
- **一括フィードバック方式**: 完成形を出し切り、問題点をユーザーが完全に把握してからまとめてフィードバックする方が全体工数が少ない、という合意。途中のやり直しリスクは許容済み
- **中間のユーザー確認は行わない**。例外は1つのみ: **サブスクリプション上限に達しそうな場合**は走行を止めて相談する
- セッション断絶(通信・PC問題)はユーザーが必ず再起動して指示を出し続ける前提。`v2-run-state.md` を常に最新化し、どのセッションからでも再開可能にする

## 参照ビジュアル

確定した参照ボードは無し。ユーザーのイメージの背景: **Age of Empires / Stronghold / Rise of Nations / 信長の野望**(踏襲義務なし、参考情報)。アートの基調は1.0で確立したペインタリー(docs/05_map-and-art 及びアート方針参照)を継続・深化。
