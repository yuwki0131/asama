## 大方針メモ

- **高低差ビジュアル**: 「1マス使い切りクリフセル」方式に移行確定(2026-07-11)
  - cliff terrain タイプを自動生成、選択不可・通行不可・建築不可
  - 崖フェイスは**シーンの深度ソートに参加**(クリフセル座標の深度キー)。#46の「常に上のオーバーレイ」は木の埋め込みを起こしたため#51で是正。建物ベース隠しは深度順で自然に成立
  - クリフセルの床は元の地形スプライトを描画(単色緑タイル問題の解消、#51)
  - 崖フェイスはレンダラーがマップ形状から導出(N高→Sフェイス、W高→Eフェイス、凹コーナーは2枚)。simのcliffFaceタグは1方向しか持てないため描画はタグ非依存(#51)
- **視覚品質基準**: Stronghold 参照 + ユーザー最終判定(実際に遊んで違和感なし)
  - サブゴール: 石垣の積み石感/高さの直感的な読み取り/天守が石垣上に乗る感
- **MAX_ELEVATION=5 (2026-07-11確定)**: Stronghold表現由来。平城・平山城(岡山/姫路)優先。6+は山城実装が別途必要(将来)。定数は変更可能な形を維持
- **地形建築制約**: 建物がある地面は盛土/削土不可(実装済み PR#43)
- **ユニットアニメ2D化 (2026-07-11ユーザー指摘)**: 3Dポリゴン感が目立つ→codexペイントオーバーで2D風に。手法=シート全体ペイントオーバー+元アルファでフレーム毎マスク(チラつき防止)。石垣h4/h5は等断面で伸ばす方式(違和感あれば台形断面に再調整)
