# MVP範囲

## MVPに含む

### 基盤

- シングルプレイヤー
- ローカル実行
- ローカルファイルへのセーブ・ロード
- 1防衛シナリオ
- 128×128アイソメマップ

### 操作・戦闘

- 個兵選択・複数選択
- 通常移動・攻撃移動・位置固定
- HP制
- 兵種相性
- 高低差補正
- 近接・弓・鉄砲
- 門破壊
- 梯子
- 堀埋め

### 城郭

- 本丸
- 柵、塀、門
- 土塁、石垣
- 空堀、水堀
- 土橋、木橋
- 櫓、天守

### 内政

- 町区画
- 農地
- 道路
- 人口
- 支持率
- 徴税
- 徴兵可能人数
- 蔵
- 市場
- 兵舎
- 武家屋敷
- 秋収穫

### 兵糧

- 防衛側蔵
- 本丸接続
- 攻撃側補給荷車
- 防衛側補給荷車
- 食料ゼロ敗北

### AI

- 正面強襲
- 梯子強襲
- 堀埋め
- 混合型

### UI

- HUD
- 建築パネル
- 詳細パネル
- ユニットパネル
- ミニマップ
- 接続・区画オーバーレイ
- 開発用デバッグステータスパネル

## MVPに含まない

- マルチプレイヤー
- 火災・火矢・放火
- 大筒・輸入大砲
- Fog of War
- 天候
- 士気
- 疲労
- 陣形維持
- 寺社
- 井戸
- 詳細物流
- 生産施設
- 技術ツリー
- 実在武将
- キャンペーン
- MOD
- 高度な外交
- 住民の個人生活シミュレーション

## MVP完成条件

1. 既存小城を改修できる
2. 町区画・農地から人口と収穫が発生する
3. 徴税・市場・徴兵が機能する
4. 本丸と蔵の接続・兵糧消費が機能する
5. 敵AIが門、梯子、堀埋めで攻撃する
6. 本丸陥落・兵糧切れで勝敗が決まる
7. セーブ・ロード後も同じ状態を復元できる

## MVP実装状況の記録(2026-07-04)

MVP完成条件1〜7はすべて実装・検証済み。追加で速度変更UI・Space一時停止・
門の開閉・シナリオのデータ駆動化(`packages/content`)・工兵(梯子・堀埋め)・
複数ユニット選択と陣形移動も実装済み。

以下は「MVPに含む」に記載があるが、初回MVP断面ではpost-MVPへ繰り延べた:

- 鉄砲兵・騎兵・補給荷車(ユニット種)
- 兵種相性・高低差・射線
- 土塁・石垣(地形/建造物の設計未確定のまま)
- 攻撃側兵糧(荷車ベース)と攻撃側兵糧切れ敗北
- ミニマップ・選択情報パネル・警報オーバーレイ
- セーブスロット複数・オートセーブ(現状はquicksave 1枠)
- ユニットの4方向表示・アニメーション、建物破壊状態スプライト
- 操作: A攻撃移動 / S停止 / H位置固定 / ダブルクリック同兵種 / グループ登録
