# Rendering

## 基本方針

描画はPixiJSで実装します。ReactはHUDとパネルを担当し、ゲーム世界の大量描画はPixiJSに寄せます。

## 座標系

シミュレーション座標はグリッド座標を正とします。

```text
cell: { x, y }
world: { x, y, z }
screen: { x, y }
```

アイソメ変換は描画層で行います。シミュレーション層は画面ピクセル座標を持ちません。

MVP初期値:

```text
tileWidth = 64
tileHeight = 32
heightStep = 8 px
```

最終タイル表示サイズは未確定事項に残しますが、MVP実装ではこの値を既定値にします。

## 描画順

基本の描画順:

1. 地形
2. 道路・農地・区画
3. 堀・橋
4. 建物基礎
5. ユニット
6. 建物上部
7. 攻城設備
8. 選択・命令・接続などのオーバーレイ
9. UI

同一レイヤー内では、原則として `x + y`、高さ、占有セル下端、実体IDの順で安定ソートします。

## カメラ

- 固定アイソメ視点
- 回転なし
- パンあり
- ズームあり

ズームは描画倍率だけを変更し、シミュレーションには影響しません。

## スナップショットと補間

Workerから受け取ったスナップショットを描画します。

ユニット位置は、前回スナップショットと今回スナップショットの間を表示上だけ補間できます。補間結果をシミュレーションへ戻してはいけません。

## アセット

MVP初期はプレースホルダー画像または単純なPixiJS図形を許容します。

正式アセットへ差し替えやすいよう、描画コードは `assetId` を参照し、ファイル名や兵種名を直接分岐条件にしません。

## オーバーレイ

MVPで必要なオーバーレイ:

- セル選択
- 移動先
- 建築予定
- 建築禁止
- 兵糧接続
- 区画
- 地形移動コスト

オーバーレイは描画層の表示責務です。接続可否、建築可否、移動コスト自体の判定はシミュレーション層の結果を使います。
