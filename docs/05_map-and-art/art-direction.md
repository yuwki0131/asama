# アートディレクション

## 基本方針

Stronghold風の固定アイソメトリック2D表現を採用します。

- 2Dプリレンダー
- カメラ回転なし
- 正投影
- 小さな個兵を多数表示
- 建物は識別性のためやや誇張

## 改訂方針(2026-07-04承認、ルック選定ゲート実施中)

「リアリティ×ゲーム的expression×アニメ的アイコン性」への改訂。4本柱:

1. シルエット第一主義: 反り(sori)のあるパラメトリック屋根、建物ごとの識別記号
2. 色彩設計の一元化: 天守基準の統一パレット。従来の「疑似ドット絵・減色調」は廃止し、クリーンなアニメ調2Dプリレンダーへ
3. アニメ的シェーディング: 固定光源による決定論的なセル/ペインタリー陰影(render-spec: `toon-cel` / `painterly`)。輪郭線は1px相当の焦茶(セル調のみ)
4. 自然環境の一級市民化: 植生キット(松・杉・森・岩・葦)、地形の濃淡、地形装飾レイヤ

ルック選定(2026-07-04確定): **C: ペインタリー**(render-spec `painterly`、全アセット supersample 2 → 縮小)。

### ハイブリッド方式(2026-07-05確定)

- **建築物・地形・接続キット** = 手続きBlender(現行)。タイル整合・接続マスク・アンカーのピクセル契約を数学的に保証できる対象はすべてこちら
- **有機的な単独装飾(木・藪など)** = **AIアート**+取り込みゲート(`docs/05_map-and-art/ai-asset-intake.md`)。手続き低ポリでは有機物の繊細さに構造的限界があるため(検証済み)。ピクセル契約が不要な単独スプライトに限る
- 一点物のヒーロー建築(天守)は既存AIアートを維持
- 取り込みゲート: 背景除去→キャンバス/アンカー正規化→天守基準へのトーン較正→接地検証(`pnpm run assets:ai-intake`)

### 品質標準(Phase B完了、ショーケース蔵で承認済み 2026-07-04)

- **トーン**: 天守AIアート実測値でキャリブレーション。屋根=暖色系の濃灰褐、漆喰=経年の暗い生成り、石=灰みの暖石色、木部=焦茶。「白すぎ・新品」を禁止し経年感を標準とする
- **瓦屋根**(`add_kawara_roof`): 桟瓦の縦列は棟沿い座標で刻み継ぎ目が流れに直交(斜行禁止)。列中央に丸瓦の捕光、目地に泥汚れ。反り(sori)カーブ、熨斗積みの大棟(黒瓦)+鬼瓦、軒付け+瓦ピッチ整列の軒丸瓦、**けらば・破風は白漆喰**
- **壁**: 塗籠(柱・長押など木部を露出させない)。雨だれの縦筋、壁裾の跳ね汚れ帯
- **蔵の識別記号**: 海鼠壁+妻面の小さな蔵紋(白丸)。建物ごとに識別記号を1つ以上
- **陰影**: 固定光方向のペインタリーランプ(暖光×青紫影)+AOの軒下・接地の落ち影
- **仕上げ**: 2倍レンダー→lanczos縮小。減色ディザは廃止済み
- 基壇の砂利パッドはフットプリント菱形と幾何学的に一致(コンタクトシートで検証)。地形との視覚的な馴染ませ縁はPhase C検討項目

## タイル

- 表示基準: 64×32px
- 制作基準: 128×64pxも可
- 最終的にゲーム内倍率で調整

## カメラ

- 固定角度
- ズーム可能
- 回転不可

## 光源

- 画面左上から固定
- 影は右下
- 昼夜変化なし
- 全アセットで同一ライティング

## 建物

- 4方向を基本
- 天守・櫓は1〜2方向でも可
- 破壊状態、瓦礫状態を用意
- 建築途中状態は不要

## 兵士

目安:

- 歩兵高さ: 24〜32px
- 騎兵: 32〜40px
- 荷車: 40〜56px

論理占有サイズは共通です。

## アニメーション

### 兵士

- 待機: 2〜4フレーム
- 歩行: 6〜8
- 攻撃: 6〜8
- 射撃: 6〜8
- 死亡: 6〜10
- 工兵作業: 6〜8
- 4方向

### 住民

- 待機
- 歩行
- 作業

## 建物内部

天守、櫓、門等にユニットが入ると、屋根・上部を半透明または非表示にします。

内部は複数階を持たず、単一の圧縮された戦闘空間として表示します。

## 色

- 彩度低め
- 土、木、瓦、漆喰の差を明確化
- 所属色は旗指物・衣服の一部
- 太い黒輪郭は避ける
- ユニットには1px程度の暗輪郭を許容
