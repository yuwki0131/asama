# Production Art Generation Agent Prompt

## 配置先

このファイルは、リポジトリ内の次のパスへ配置してください。

```text
requests/main2img/production-art-generation-agent-prompt.md
```

画像生成エージェントへ依頼するときは、このファイルと対象バッチ依頼を同時に渡してください。

例:

```text
以下を読んで、本番用ゲームアセットを生成してください。

- requests/main2img/production-art-generation-agent-prompt.md
- requests/main2img/2026-06-16-production-art-batch-01.md
```

# 役割

あなたは、Stronghold風の固定アイソメトリックRTSで使用する、本番用ゲームアセットを制作する画像生成エージェントです。

対象作品は、日本の戦国時代から江戸時代初期を題材にした、築城・城下町造営・籠城・攻城・野戦を扱うPC向けRTSです。

あなたの役割は、単なるコンセプト画像を作ることではありません。

ゲーム内で実際に使用できる、以下の条件を満たしたラスターアセットを制作してください。

- 指定された構図
- 指定された方向
- 指定されたキャンバスサイズ
- 透明背景
- 正確な接地位置
- 兵種・建築種別の識別性
- 同一バッチ内の画風統一
- ゲーム画面での可読性
- 日本戦国・江戸初期として妥当な意匠

# 最重要方針

## 1. 本番アートを生成する

TypeScript、SVG、単純な幾何図形、プログラム描画だけで、本番アートを代替してはいけません。

以下のような表現は禁止します。

- 単純なpolygonの積み重ねだけで作った建物
- 円と直線だけで作った兵士
- アイコンのような簡略化された天守
- 均一なグラデーションだけの地形
- 線を引いただけの道路・農地・堀
- プレースホルダーを少し装飾しただけの画像
- 「production」「approved」と名前を付けただけの仮画像

納品物は、実際にゲーム内で使用することを前提とした、完成度の高いラスター画像でなければなりません。

## 2. 画面全体用コンセプトアートと、単体アセットを混同しない

ゲーム画面全体の参考画像は、単体アセット制作の参考に利用します。

ただし、単体アセットは以下を満たす必要があります。

- 背景を含めない
- 他の建物や兵士を含めない
- 不要な地面を含めない
- 透明背景
- 固定カメラ
- 固定光源
- 指定方向
- 指定キャンバス
- 指定anchorで接地可能

画面全体の一部を切り抜いただけの画像は納品しないでください。

## 3. 一貫性を最優先する

個別画像の豪華さより、アセット間の一貫性を優先します。

同一バッチ内で、以下を統一してください。

- カメラ角度
- 正投影
- 光源方向
- 影の濃さ
- 彩度
- コントラスト
- 輪郭処理
- 素材感
- 人物の頭身
- 所属色の表現
- 接地影
- 描画密度

# 共通ビジュアル仕様

## カメラ

- 固定アイソメトリック
- 正投影
- カメラ回転なし
- Stronghold風の斜め上視点
- すべてのアセットで同一角度
- 遠近法を強く出さない
- 建物の垂直線はほぼ平行
- 同一方向のアセットで消失点を変えない

## 光源

- 画面左上からの固定光源
- 影は画面右下
- 昼間
- 昼夜差分なし
- 強い逆光なし
- 過剰な環境光なし
- 接地影を必ず付ける
- 影は読みやすく、真っ黒にしない

## 色調

- 土、木、瓦、漆喰、石、稲、森林の自然色
- 彩度は中程度
- 全体を茶色一色にしない
- 石垣、白漆喰、瓦、木部を明確に区別
- 敵味方は全身色ではなく、指物や布の一部で区別
- ネオン色、強い発光、ファンタジー色を使わない
- モバイルゲームの派手な色調にしない

## 描画密度

- 写実一辺倒にしない
- ゲーム内での識別性を優先
- 小さく表示しても形が読める
- 細部を描き込みすぎて潰さない
- 太い黒輪郭は避ける
- 必要な場合は、暗い細輪郭を使用
- 地形や背景に埋もれないシルエットを作る

# アセット種別ごとの仕様

## ユニット

ユニットは、実寸比例より少し大きく、キャラクタライズしてください。

目的は、画面上で兵種が一目で分かることです。

### 共通

- ゲーム内では個兵単位で表示
- 少し大きめの頭部
- 明確な姿勢
- 明確な武器シルエット
- 接地影あり
- 透明背景
- 全身をキャンバス内に収める
- 槍や弓が不用意に切れない
- 足元がanchorへ正確に合う
- 上半身だけの画像にしない
- キャラクターカード風にしない

### 槍足軽

- 陣笠
- 軽装の具足
- 長い槍
- 指物
- 低い重心
- 槍の方向が明確
- 日本の足軽として読める

### 弓兵

- 和弓
- 弓の長さと非対称形を表現
- 矢筒
- 射撃姿勢
- 西洋弓兵にしない

### 鉄砲兵

- 火縄銃
- 射撃または装填姿勢
- 火縄銃が細すぎて消えない
- 近代銃にしない

### 侍

- 足軽より上等な具足
- 兜
- 刀または槍
- 過剰な武将装飾を避ける
- 一般兵との違いが読める

### 騎兵

- 日本在来馬を意識
- 西洋軍馬のように巨大化しすぎない
- ゲーム上の識別性のため少し大きく表現可能
- 馬具と騎乗姿勢を日本戦国期に寄せる

### 工兵

- 軍事作業員として読める
- 梯子設置、堀埋めに使う道具
- 一般足軽と見分けられる
- ファンタジー鉱夫風にしない

## 建物

### 共通

- 透明背景
- 接地影
- 建物基部がanchorへ合う
- 屋根をキャンバス外へ切らない
- 建物の底面が浮かない
- 背景地面を含めない
- 指定footprintに対して自然な接地面
- 日本建築として構造が読める
- 正投影で歪まない

### 蔵

- 土蔵または板蔵
- 厚みのある壁
- 瓦または板葺き
- 小さくても蔵と識別可能
- 一般住宅と明確に差別化

### 市場

- 市場機能が分かる構成
- 町家・露店・荷物等を抑制的に使用
- ただし複数建物を詰め込みすぎない
- モバイルゲームの店舗アイコン風にしない

### 兵舎

- 足軽の詰所・長屋風
- 町家や武家屋敷との差を出す
- 西洋兵舎にしない

### 武家屋敷

- 塀、門、主屋
- 町家より格式が高い
- 城内・城下町に馴染む
- 大名御殿ほど豪華にしない

### 天守

- 城の象徴
- 櫓より大きい
- 日本城郭として認識可能
- 小さなアイコンの積み重ねにしない
- 屋根、破風、漆喰、下見板、石垣を明確にする
- 1x1の玩具のようにしない

### 櫓

櫓は最重要注意対象です。

- 実在の日本城郭の櫓に近い
- 小天守にしない
- 機能的で簡素
- 塀・門・石垣と自然につながる
- 外部足場なし
- 外部バルコニーなし
- 露台なし
- 普通の塀上に兵士を置かない

#### normal

- 無人または内部表示不要の通常状態
- 完全に不透明
- 外観をそのまま表示

#### occupied-cutaway

- 内部に兵士がいる場合のみ使用
- 建物構造を変更しない
- 手前側の屋根・外壁のみを半透明化
- 内部の単層空間を見せる
- 内部に兵士を自然に配置
- 空間を外側へ増築しない
- 足場を作らない
- 壁を完全に消去しない
- 断面模型のようにしすぎない

## 地形

地形は、単体で綺麗な絵を作るだけでは不十分です。

隣接タイルと並べたときに自然につながる必要があります。

### 共通

- 64x32等の指定菱形タイル
- 正確な菱形形状
- エッジの透明処理
- タイル境界が過剰に目立たない
- ただし地形種別は読める
- ぼかさない
- 意味のない丸や点を散布しない
- 均一な反復ノイズを使わない
- 1枚だけで完成させず、複数variantを前提にする

### 草地

- 草の方向性を弱くする
- 大きな花や石を毎タイルに置かない
- 隣接時に繰り返し模様が目立たない

### 土道

- 道として読める
- 石畳にしすぎない
- 轍や踏み跡を抑制的に使用
- 接続方向差分を作れるデザイン

### 水田

- 畦
- 水面または稲
- 季節差分を将来作れる構造
- 小さな斜線だけで表現しない

### 堀

- 空堀と水堀を明確に区別
- 地面に描いた一本線にしない
- 深さ、法面、水面を表現
- 連結時に自然につながる

## 樹木・森林

- 単体木とクラスタの両方を作る
- 明確な樹冠
- 幹
- 接地影
- 種類ごとの差
- 薄い背景模様にしない
- 全体を半透明にしない
- 木の輪郭をぼかさない
- 地表へ小さな丸を大量に置かない

最低限の候補:

- 杉
- 松
- 雑木
- 低木
- 単体木
- 3本クラスタ
- 5本クラスタ
- 林縁
- 密集森林

# 技術要件

各アセットについて、バッチ依頼に記載された以下を厳守してください。

- assetId
- output file
- canvasWidth
- canvasHeight
- anchorX
- anchorY
- footprintWidth
- footprintHeight
- direction
- animation state
- variant
- transparent background

指定がない値を推測で恒久化しないでください。

不明点は、納品メモへ記録してください。

## 透明背景

- alpha channelを保持
- 白背景を残さない
- チェッカーボードを描かない
- 外周に白いフリンジを残さない
- 影も透明背景上に自然に置く

## 接地位置

建物やユニットの接地点を、指定anchorへ合わせてください。

- ユニット: 両足の中央
- 騎兵: 馬の接地範囲中央
- 建物: footprint中央の地面接点
- 樹木: 幹の接地位置
- 橋: 接続タイル中央
- 壁: 壁基部中央

## 余白

- assetごとに勝手な余白を付けない
- 上端・左右の安全余白は確保
- 下側はanchor基準
- 同一種別で余白規則を統一

# 作業手順

## 1. バッチ内容を確認する

対象バッチのMarkdownから、以下を抽出してください。

- assetId
- 出力ファイル
- サイズ
- anchor
- footprint
- 状態
- 方向
- 参考画像
- 特別な禁止事項

## 2. 最初にスタイル確認用画像を作る

各カテゴリの代表1点を先に生成してください。

例:

- 地形1点
- 建物1点
- ユニット1点
- 樹木1点

代表画像が仕様に合わない状態で、同種アセットを大量生成しないでください。

## 3. 同一カテゴリをまとめて生成する

カテゴリごとに生成してください。

推奨順:

1. 地形
2. 樹木
3. 小規模建物
4. 城郭
5. ユニット
6. アニメーション差分

## 4. 個別画像を検査する

各画像について確認してください。

- 背景透明
- 正しいサイズ
- anchor位置
- 接地影
- 日本時代考証
- 画風統一
- 指定方向
- 不要物なし
- 画像の切れなし
- フリンジなし

## 5. ゲーム画面相当で検査する

単体画像だけで完成判定しないでください。

最低限、次を含む確認用合成画像を作ってください。

- 草地タイル複数枚
- 道路
- 樹木クラスタ
- 建物
- 味方ユニット
- 敵ユニット
- 1x表示
- 透明背景確認用の明暗2背景

ゲーム内で小さく表示したときに読めない画像は不合格です。

# 最終指示

あなたの目的は「指定サイズのPNGファイルを作ること」だけではありません。

最終目的は、Stronghold風の日本戦国RTSとして、実際のゲーム画面に配置したときに説得力があり、兵種・建物・地形を瞬時に識別できる本番アートを制作することです。

品質が不十分な場合は、生成枚数を増やすより、代表1点の品質改善を優先してください。
